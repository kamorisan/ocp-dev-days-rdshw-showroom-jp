== あなたの役割とミッション

あなたは **Parasol Insurance** の新しい開発者です。

Parasol は急速に成長しています。新しい製品ライン、新しいチーム、新しい開発者 -- しかし、プラットフォームチームは追いつくことができません。新しいチームごとにカスタム CI/CD セットアップ、独自の namespace プロビジョニング、数週間にわたるオンボーディングプロセスが必要です。開発者はコードを書くよりも、チケットを提出してアクセスを待つことに多くの時間を費やしています。ローカル環境が頻繁に壊れたり変化したりして、生産性が失われています。時々オンボードする契約社員は、関連情報やベストプラクティスを見つけるのに長い時間がかかります。

聞き覚えがありますか?

Parasol のプラットフォームエンジニアリングチームは解決策に取り組んできました: 開発者ツール、CI/CD、GitOps 駆動のデプロイメント、セルフサービス型の内部開発者ポータルをすべて 1 か所で提供する、Red Hat OpenShift 上に構築された統合アプリケーションプラットフォームです。

**今日があなたの初日です。** あなたはプラットフォームへのアクセスを与えられ、最初のタスクを割り当てられました。このワークショップの終わりまでに、1 つのチケットも提出せずに本番環境に機能をリリースすることになります。

== 達成すること

このワークショップの終わりまでに、以下の実践的な経験を得られます:

* **Red Hat Developer Hub** -- ソフトウェアカタログをナビゲートし、API とドキュメントを発見し、AI アシスタントを使用してプラットフォームに関する回答を得る
* **セルフサービステンプレート** -- ゴールデンパステンプレートを使用して、独自の CI/CD パイプラインを持つ分離された開発環境を作成する
* **Red Hat OpenShift Dev Spaces** -- ローカルに何もインストールせずに、AI コーディングアシスタントを備えた事前設定されたクラウド IDE でコードを書く
* **OpenShift Pipelines** -- コードをプッシュして、自動化された CI/CD パイプラインが変更をビルド、テスト、スキャンするのを確認する
* **OpenShift GitOps** -- Argo CD が変更を本番環境に自動的にデプロイするのを確認する
* **OpenShift Console** -- より深いデバッグとアプリケーションモニタリングのために Web コンソールを使用する

== 開発者ジャーニー

このワークショップは、1 つの連続したストーリーに従います:

. **プラットフォームとの出会い** -- Developer Hub にログインし、ソフトウェアカタログを探索し、オリエンテーションを受ける
. **あなたのミッション** -- 割り当てられたタスクを見つけ、開発環境をセットアップする
. **機能を構築する** -- Dev Spaces を開き、コードを書き、ローカルでテストする
. **リリース** -- CI/CD パイプラインを通じてコードをプッシュし、品質ゲートに到達する
. **AI を呼び出す** -- エージェント型 AI アシスタントを使用してコードを修正し、再度プッシュし、パイプラインを通過する
. **本番環境へ** -- 変更をマージし、リリースを作成し、本番環境で機能を確認する

切り離されたツール間のコンテキストスイッチングはありません。アイデアから本番環境まで 1 つの連続したフローです。

== 対象者

このワークショップは以下の方向けに設計されています:

* より速く、より一貫性のある開発ワークフローを求めるソフトウェア開発者
* アプリケーションプラットフォーム機能を評価する IT 意思決定者
* 自分たちが可能にしている開発者エクスペリエンスを理解したいプラットフォームエンジニア
* OpenShift の開発者ツールの実践的な経験に興味がある方

== 前提条件

以下が必要です:

* Web アプリケーションと API に関する基本的な知識
* Web ブラウザとコードエディタの使用に慣れていること
* 学ぶ意欲 -- OpenShift やコンテナの事前経験は不要
